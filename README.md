# 🐍 Python for Electrical Engineering & IT

Dieses Repository dokumentiert meinen Fortschritt im Modul **Python** an der Technischen Hochschule Ingolstadt. Ziel des Kurses ist es, die Sprache von den Grundlagen bis hin zu fortgeschrittenen softwaretechnischen Konzepten zu beherrschen.

## 📂 Aufgabenübersicht

Hier findest du die Lösungen zu den wöchentlichen Aufgabenpaketen:

* **[Task 01: Werte & Datentypen](./task_01_datatypes/)** 🔢
  * Einführung in Variablen, Strings und grundlegende Datenstrukturen.
* **[Task 02: Zahlen-Analyse-Tool](./task_02_analysis/)** 📊
  * Logik-Implementierung zur Auswertung numerischer Datenreihen.
* **[Task 03: Conway Folge](./task_03_conway/)** 🧬
  * Algorithmische Umsetzung der "Look-and-Say"-Sequenz.
* **[Task 04: Operator Überladung](./task_04_overloading/)** ⚙️
  * Objektorientierte Programmierung: Anpassung von Standard-Operatoren für eigene Klassen.
* **[Task 05: Verkettete Liste](./task_05_linked_list/)** 🔗
  * Implementierung einer dynamischen Datenstruktur (Linked List) "from scratch".
* **[Task 06: Lottozahlen](./task_06_lotto/)** 🎰
  * Zufallsalgorithmen, Listen-Handling und Validierung.
* **[Task 07: Coming Soon...](#)** 🚀
  * (Wird nach Bekanntgabe ergänzt).

## 🛠️ Technologien & Tools
* **Sprache:** Python 3.10+
* **Methodik:** Test-Driven Development (TDD) Ansätze mit separaten Test-Files.
* **Umgebung:** Visual Studio Code / GitHub Web Interface.

---
## 🔍 Über dieses Repository
Die Struktur folgt dem Prinzip der sauberen Trennung (Separation of Concerns). Jeder Task enthält:
1. Eine eigene `README.md` mit der Aufgabenstellung.
2. Die Kern-Logik als `.py` Datei.
3. Eine `test_*.py` Datei zur Verifizierung der Anforderungen.
